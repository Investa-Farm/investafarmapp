# 🚜 Investa Farm Platform Build Instructions

Build a production-ready agricultural finance platform called **Investa Farm**.

The platform combines:

1. Satellite-based farm monitoring
2. Farmer wallet infrastructure using Stellar SDK
3. Farm credit scoring
4. Input financing management
5. Farmer and farm management

The system should be modular and scalable.

---

# 📡 SATELLITE INTELLIGENCE MODULE

## Objective

Use free satellite imagery to monitor farms and generate farm health scores.

### Data Sources

Use:

* Sentinel-2 imagery for crop health
* Google Earth Engine as the processing layer

### Required Functionality

#### Farm Registration

Farmers can register:

* Farm name
* Farm size
* Crop type
* GPS coordinates
* Farm boundary polygon

Store farm boundaries in the database.

---

#### NDVI Calculation

When a farm is registered:

1. Connect to Google Earth Engine.
2. Retrieve Sentinel-2 imagery.
3. Filter imagery:

   * By farm boundary
   * By date
   * Cloud coverage less than 20%

Calculate:

```text
NDVI = (B8 - B4) / (B8 + B4)
```

Store:

* Average NDVI
* Latest NDVI
* Historical NDVI values

---

#### Farm Health Classification

Classify farms:

```text
NDVI > 0.60 = Healthy

0.30 – 0.60 = Moderate

Below 0.30 = High Risk
```

---

#### Farm Dashboard

Display:

* Current NDVI
* NDVI trend chart
* Health status
* Risk status
* Last satellite update

# 💰 STELLAR WALLET MODULE

## Objective

Use Stellar SDK as the wallet infrastructure.

Do NOT use:

* Mainnet
* Testnet
* Smart contracts
* Token deployment

Use Stellar only for wallet creation and wallet identity.

---

### Wallet Creation

For every farmer:

Generate:

* Stellar public key
* Stellar secret key

Using:

```python
from stellar_sdk import Keypair

keypair = Keypair.random()
```

Store:

* Public key
* Encrypted secret key

---

### Internal Wallet Ledger

Balances should be stored internally.

Example:

```text
Farmer Wallet

Balance: 25,000 KES
Credit Limit: 50,000 KES
Available Credit: 25,000 KES
```

Do not depend on blockchain transactions.

Use PostgreSQL.

---

### Transaction Types

Support:

* Loan disbursement
* Input financing
* Harvest repayment
* Wallet transfer
* Bonus rewards

Store full transaction history.
