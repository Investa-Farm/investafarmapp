-dive. I've organized everything by priority and role so you can see exactly what's missing and what matters most.

🔴 Critical — Broken or Missing Core Flows
1. Pending Investment "Resume" Banner
When an investor taps Invest → goes to fund wallet → comes back, the pending investment is saved in sessionStorage but there's no "Complete your purchase" prompt to bring them back. They have to remember to click Invest again. Many users will just abandon.

2. Portfolio Holdings — No Visual Status Badges
After an investor requests an exit or lists shares for sale, the holding card in their portfolio looks exactly the same as before. There's no "🔄 Exit Pending" or "📋 Listed for Sale" overlay. Users don't know if their request went through.

3. Farmer KYC Rejection — Silent Failure
If admin rejects a farmer's KYC document, the farmer only sees a status change deep in the KYC page. The dashboard banner is binary (verified or not) — it doesn't say which document was rejected or why. Farmers get stuck not knowing what to fix.

4. B2C Payouts Are Simulated
Card and USDC withdrawals are debited from the wallet and stuck in processing status forever — no actual money movement happens. Users who initiate these will never receive funds.

5. Agribusiness KYC Has No Hard Block
Investors get blocked from trading without KYC. But agribusiness users have no equivalent enforcement — they can use the dashboard without uploading documents.

6. Commission Earned Always Shows Zero
The /api/agribusiness/stats endpoint hardcodes commissionEarned: 0. Every agribusiness/sales agent dashboard shows KES 0 commission regardless of actual activity.

🟠 High Priority — Significant UX Gaps
7. No Harvest Report Submission Form
When a farm reaches harvest, there's no dedicated form for the farmer to submit actual yield vs. expected yield. Everything relies on admin manually entering it or the farmer posting a generic update. This creates a trust gap with investors.

8. No Supplier Map / Nearest Dealer
Farmers redeeming input vouchers see a flat list of agro-dealers but have no map or "nearest to me" search. In rural Kenya this is a critical missing feature — farmers may not know supplier locations.

9. No "Disaster / Crop Failure" Reporting Flow
The loan contract mentions insurance and restructuring, but there's no dedicated flow for a farmer to report crop failure and trigger a formal response. They can only post a generic update.

10. Secondary Market Order Book UX
The order book uses raw trading terms (Bid/Ask, Limit Price, Depth) with no explainer tooltips or simplified mode. Most Kenyan smallholder investors will be confused. Needs a "Simple Mode" vs "Advanced Mode" toggle.

11. Withdrawal Has No Progress Tracker
After a farmer or investor initiates M-Pesa withdrawal, the UI just says "1-2 business days." There's no status tracker, no SMS confirmation echo, and no "cancel if pending" option.

12. Resend OTP Cooldown Resets on Refresh
The 60-second OTP resend cooldown lives only in component state — refreshing the page resets it to 60s, allowing users to spam the resend button and the email/SMS provider.

13. Rate Limit Error Shows "Connection error"
When login is rate-limited (429), users see a generic "Connection error" instead of a helpful "Too many attempts — try again in X minutes" message. The API already sends retryAfterMs but the frontend ignores it.

🟡 Medium Priority — Missing Features Users Will Expect
14. No "Complete Your Purchase" Notification
Related to #1 — investors who added funds separately have no way to know they have a pending investment. A sticky banner or notification ("You were investing in Kilimo Maize Farm — complete your purchase →") would massively improve conversion.

15. Farmer Has No "Repay Early" or Crop Failure Path
The only repayment mechanism is automatic harvest split. There's no manual early repayment UI and no way to acknowledge/document crop failure formally.

16. Offtaker Market Prices API Mismatch
The offtaker dashboard calls /api/offtaker/market-prices which doesn't exist in the routes. It likely falls through to a 404 — offtakers see a broken Prices tab.

17. Cooperative Route Optimization Is Mocked
The AI route optimizer in the cooperative dashboard uses MOCK_DELIVERIES — no real delivery data from the backend. Real cooperatives using this will get fake routes.

18. No Wallet Receipt / Transaction Detail Screen
Tapping a transaction in the wallet history does nothing meaningful. There's a receiptTx state in the modal but the receipt modal is empty — no reference number, breakdown, or download.

19. Admin — No Segment-Based Messaging
The admin can broadcast to everyone or message individuals, but can't message a specific segment (e.g., "all farmers in Nakuru whose KYC is pending"). This is critical for targeted operational communication.

20. Admin — No Risk / Fraud Flags
There are no automated alerts for suspicious activity like multiple accounts on one IP, circular fund transfers, or unusually large single transactions. These need at minimum a flagging system in the Overview tab.

21. GPS / Location Step Missing from Farmer KYC UI
The backend has a /api/kyc/location endpoint for capturing GPS coordinates, but the KYC page never calls it. Location verification is silently skipped for all farmers.

22. No "Magic Link" Fallback for OTP Verification
If a user can't receive OTP (SMS/email issue), there's no fallback verification method. They're completely locked out with no alternative path.

🔵 Polish & UX Improvements
23. Investor Swap Feature is Incomplete
The Portfolio has a "Swap" button but there's no proper swap modal — users hit a dead end or see a simplified placeholder.

24. Camera Permission Denied — No Graceful Fallback
In investor KYC, if the user denies camera permission, the UI gets stuck. There's no immediate "Upload selfie from gallery instead" fallback.

25. Admin — No Audit Log Export
The login audit log is visible but there's no export button — compliance/legal teams can't download it.

26. Admin — No 2FA Enforcement for Sub-Admins
Farmers can set up TOTP but admin sub-accounts have no option to require 2FA. A compromised viewer account can read all user financial data.

27. Farmer Operations — No "Days Since Last Update" Warning
There's no prompt reminding farmers to post updates. Investors lose confidence when updates stop — a "Post an update (last posted X days ago)" nudge would help retention.

28. No Dark Mode on Landing / Onboarding
The onboarding slides and landing page have no dark mode support, creating a jarring flash when a user with system dark mode enabled first opens the app
