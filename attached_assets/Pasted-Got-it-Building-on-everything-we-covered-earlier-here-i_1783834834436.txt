1. Core Purpose of the Secondary Market

The secondary market allows investors to buy and sell existing farm shares before harvest. This provides:

· Liquidity – investors can exit early if they need cash.
· Price discovery – share prices reflect real‑time supply/demand and changing farm conditions.
· Arbitrage opportunities – sophisticated investors can profit from mispriced shares.

3. The Pricing Engine (Real‑Time Fair Value)

This AI calculates the current fair value of a share based on the farm’s updated forecast revenue, risk score, and market conditions (same formula we derived earlier).

Endpoint: GET /api/secondary/fair-value/{farm_id}

Formula:

P_{\text{fair}} = \frac{\alpha \hat{R}(t)}{N (1 + r_f)^{(T-t)/365}} \times (1 - \lambda(t)) \times (1 + \beta \cdot \text{imbalance}) \times (1 - \delta)

Where:

· \hat{R}(t) = AI‑updated revenue forecast (from satellite/weather inputs)
· S(t) = AI‑updated risk score
· \lambda(t) = risk discount = \frac{10 - S(t)}{9} \times p_{\max} \times L
· \text{imbalance} = \frac{\text{total bid volume} - \text{total ask volume}}{\text{total shares outstanding}}
· \beta = demand‑supply sensitivity (e.g., 0.2)
· \delta = liquidity discount (e.g., 0.02 for 2%)

Implementation for Replit:

```python
def calculate_fair_value(farm_id, order_book_db):
    # Fetch farm from MongoDB
    farm = db.farms.find_one({"_id": farm_id})
    if not farm:
        return None
    
    # Get current order book volumes for this farm
    bid_volume = db.order_book.aggregate([
        {"$match": {"farm_id": farm_id, "side": "buy", "status": "open"}},
        {"$group": {"_id": None, "total": {"$sum": "$quantity"}}}
    ])
    ask_volume = db.order_book.aggregate([
        {"$match": {"farm_id": farm_id, "side": "sell", "status": "open"}},
        {"$group": {"_id": None, "total": {"$sum": "$quantity"}}}
    ])
    
    Q_bid = next(bid_volume, {}).get('total', 0)
    Q_ask = next(ask_volume, {}).get('total', 0)
    Q_total = farm['total_shares']
    
    # Time remaining
    now = datetime.utcnow()
    days_left = (farm['harvest_date'] - now).days
    if days_left < 0:
        days_left = 0
    
    # Risk discount
    S = farm.get('risk_score', 5)
    p_max = farm.get('p_max', 0.4)
    L = farm.get('loss_given_default', 0.8)
    lambda_risk = ((10 - S) / 9.0) * p_max * L
    
    # Time discount
    r_f = farm.get('risk_free_rate', 0.10)
    time_factor = (1 + r_f) ** (days_left / 365.0)
    d = (1 - lambda_risk) / time_factor
    
    # Expected dividend
    alpha = farm['investor_share']
    R_hat = farm['forecast_revenue_usd']
    N = farm['total_shares']
    expected_div = (alpha * R_hat) / N
    
    # Fair value before market adjustments
    fair_base = expected_div * d
    
    # Imbalance adjustment
    if Q_total > 0:
        imbalance = (Q_bid - Q_ask) / Q_total
    else:
        imbalance = 0
    demand_multiplier = 1 + 0.2 * imbalance
    
    # Liquidity discount (seller price)
    liquidity_discount = 0.02
    seller_price = fair_base * demand_multiplier * (1 - liquidity_discount)
    buyer_price = fair_base * demand_multiplier * (1 + liquidity_discount)
    
    return {
        "fair_value_base": round(fair_base, 4),
        "seller_price": round(seller_price, 4),
        "buyer_price": round(buyer_price, 4),
        "mid_price": round((seller_price + buyer_price) / 2, 4),
        "risk_discount": round(lambda_risk, 4),
        "days_to_harvest": days_left,
        "imbalance_factor": round(demand_multiplier, 4)
    }
```

4. The Matching Engine (Order Execution)

This is the heart of the secondary market. It runs continuously (or every few seconds) to match open buy and sell orders.

Algorithm (Price‑Time Priority):

1. Fetch all open buy orders for a farm, sorted by limit_price descending (highest bid first), then created_at ascending (oldest first).
2. Fetch all open sell orders, sorted by limit_price ascending (lowest ask first), then created_at ascending.
3. While the highest bid ≥ lowest ask:
   · Trade quantity = min(bid.quantity, ask.quantity)
   · Execution price = ask.limit_price (seller sets the minimum they'll accept)
   · Deduct quantity from both orders.
   · If quantity becomes 0, mark order as filled; else mark as partial.
   · Record the trade in a trades collection.
   · Trigger settlement (see next section).
   · Move to the next order(s).

Matching Engine Implementation for Replit:

```python
def match_orders(farm_id):
    db = get_db()
    
    # Get best bids (highest price first)
    bids = list(db.order_book.find(
        {"farm_id": farm_id, "side": "buy", "status": "open"}
    ).sort([("limit_price_usd", -1), ("created_at", 1)]))
    
    # Get best asks (lowest price first)
    asks = list(db.order_book.find(
        {"farm_id": farm_id, "side": "sell", "status": "open"}
    ).sort([("limit_price_usd", 1), ("created_at", 1)]))
    
    i, j = 0, 0
    trades = []
    
    while i < len(bids) and j < len(asks):
        bid = bids[i]
        ask = asks[j]
        
        # Check if they can cross
        if bid['limit_price_usd'] < ask['limit_price_usd']:
            break  # No more matches possible
        
        # Execute trade
        qty = min(bid['quantity'], ask['quantity'])
        price = ask['limit_price_usd']  # Seller's ask price
        
        # Update bid
        new_bid_qty = bid['quantity'] - qty
        if new_bid_qty <= 0.0001:
            db.order_book.update_one({"_id": bid['_id']}, {"$set": {"status": "filled"}})
        else:
            db.order_book.update_one({"_id": bid['_id']}, {"$set": {"quantity": new_bid_qty, "status": "partial"}})
        
        # Update ask
        new_ask_qty = ask['quantity'] - qty
        if new_ask_qty <= 0.0001:
            db.order_book.update_one({"_id": ask['_id']}, {"$set": {"status": "filled"}})
        else:
            db.order_book.update_one({"_id": ask['_id']}, {"$set": {"quantity": new_ask_qty, "status": "partial"}})
        
        # Record trade
        trade = {
            "farm_id": farm_id,
            "buyer_id": bid['investor_id'],
            "seller_id": ask['investor_id'],
            "quantity": qty,
            "price_usd": price,
            "total_usd": qty * price,
            "buyer_fee": qty * price * 0.005,
            "seller_fee": qty * price * 0.005,
            "created_at": datetime.utcnow()
        }
        db.trades.insert_one(trade)
        trades.append(trade)
        
        # Move to next orders if fully filled
        if new_bid_qty <= 0.0001:
            i += 1
        if new_ask_qty <= 0.0001:
            j += 1
    
    # Return summary
    return {"executed_trades": len(trades), "total_volume": sum(t['total_usd'] for t in trades)}
```

5. Stellar Settlement (Atomic Swap)

Once a trade is matched, you need to settle it on the Stellar network. This is a two‑step atomic transfer (buyer pays USDC → seller receives USDC, buyer receives FARM tokens → seller loses FARM tokens). You can do this with a single path payment or two separate transactions.

Simpler approach (two transactions) for Replit:

1. Buyer pays seller:
   payment from buyer's Stellar account to seller's Stellar account, amount = total USD (in USDC).
2. Transfer FARM tokens:
   payment of the FARM asset from seller to buyer.

Platform fee collection: On both payments, deduct a small fee to the platform's Stellar account.

What to tell Replit:

After a trade is matched and recorded in MongoDB, execute settlement on Stellar:

1. Check that both buyer and seller have established trustlines to the farm's asset issuer.
2. Create a Stellar transaction with two operations:
   · payment of USDC from buyer → seller (amount = trade.total_usd minus seller's fee).
   · payment of FARM asset from seller → buyer (quantity = trade.quantity).
3. Submit the transaction to the Stellar network.
4. On success, update the trade record with the Stellar transaction hash.
5. On failure, revert the MongoDB trade status (mark as pending, retry or notify support).

6. The No‑Arbitrage AI Check

This is an automated watchdog that compares the current market mid‑price with the fair value derived from the AI pricing engine. If the deviation exceeds a threshold (e.g., 5%), the platform can:

· Send an alert to investors (arbitrage opportunity).
· Adjust the liquidity discount dynamically to bring it back in line.
· Flag the farm for manual review.

Implementation:

```python
def check_arbitrage(farm_id):
    db = get_db()
    
    # Get fair value from AI
    fair = calculate_fair_value(farm_id, db.order_book)
    
    # Get current best bid and ask from order book
    best_bid = db.order_book.find_one(
        {"farm_id": farm_id, "side": "buy", "status": "open"},
        sort=[("limit_price_usd", -1)]
    )
    best_ask = db.order_book.find_one(
        {"farm_id": farm_id, "side": "sell", "status": "open"},
        sort=[("limit_price_usd", 1)]
    )
    
    if not best_bid or not best_ask:
        return {"status": "insufficient_liquidity"}
    
    mid_market = (best_bid['limit_price_usd'] + best_ask['limit_price_usd']) / 2
    fair_value = fair['mid_price']
    
    deviation = (mid_market - fair_value) / fair_value * 100  # percentage
    
    return {
        "mid_market": mid_market,
        "fair_value": fair_value,
        "deviation_percent": round(deviation, 2),
        "arbitrage_opportunity": abs(deviation) > 5,
        "signal": "BUY" if deviation < -5 else ("SELL" if deviation > 5 else "HOLD")
    }
```
