🔴 High priority (breaks flow)

OTP lock-in — If a user closes the OTP screen mid-login, they have to re-enter their full password to get another code. There's no "Resend OTP" entry point from the login screen itself.
GPS hang on farmer registration — The GPS capture step has no timeout. If the farmer is indoors or in a low-signal area, the spinner hangs indefinitely with no "Enter manually" fallback. Rural Kenya = frequent signal issues.
KYC selfie failure silence — If the user denies camera permission, the UI falls back but gives no actionable instruction ("Go to browser settings → allow camera"). Users are left confused.
Wallet → Invest dead-end — From the Invest modal, tapping "Go to Wallet" navigates away and loses the farm the user was about to invest in. When they return to the market, they have to find that farm again.
🟡 Medium priority (friction / confusion)

Onboarding has no role split — The 3-slide splash is the same for farmers and investors. Since their whole app experience is different, a simple "I'm a Farmer / I'm an Investor" branch on the last slide would set the right expectation before registration.
Investor Checklist can't be restored — Once dismissed, there's no way to bring back the onboarding checklist. Users who dismiss it early (before completing KYC) lose their progress guide permanently.
Portfolio is too dense on small screens — Header + balance + 3-stat row + tabs + chart + period toggle + holdings list = 7 stacked sections before you even reach your actual investments. On iPhone SE-sized screens this is painful.
Empty Farm Feed tab — The Activity page's "Farm Feed" tab shows nothing until you have an active investment. There's no empty state explaining why it's blank — new users think it's broken.
Farmer dashboard "Apply for Funding" is buried — The single most important action for a new farmer is hidden below the wallet section. If the farmer has no active crop, this should be a full-screen empty-state CTA.
Theme toggle is in the wrong place — The dark/light mode icon sits in the farmer dashboard header. Users expect it in Profile → Settings, not as a primary header action alongside "Notifications."
🟢 Lower priority (polish)

No back button on Onboarding slides — Swiping back works but isn't obvious. Adding a visible "← Back" or back arrow makes it accessible to non-swipe users.
KYC location form is one long scroll — County, Sub-County, Ward, Landmark, Physical Address, GPS is six fields on one form inside a modal. Splitting into 2-step (Location → GPS) would increase completion rates.
Notification badge shows nothing on API failure — If the notification API call fails, the badge just disappears silently. A neutral "•" indicator would be better than nothing.
Secondary market 3-tab layout needs visual hierarchy — "All Listings / Order Book / Mine" tabs look identical in weight. "Mine" should have a count badge, and "Order Book" should feel more technical/different since it's a different concept.
