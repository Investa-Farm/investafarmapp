### CURRENT ISSUE

The project is a monorepo with:

* Frontend: artifacts/investa-farm (React 19 + Vite 7 PWA)
* Backend: artifacts/api-server (Express 5 API server)

Vercel is currently misconfigured and attempting to build the backend, which is causing deployment failures (no public/output directory, server not compatible with Vercel runtime).

## REQUIRED RESTRUCTURE

### 1. FRONTEND (Vercel deployment target)

Ensure the frontend is the ONLY Vercel-deployable app:

Path:

```
artifacts/investa-farm
```

Confirm:

* It has a Vite build system
* package.json includes:
  "build": "vite build"
* Output directory is: dist

---

### 2. BACKEND (NOT deployed on Vercel as Express server)

Keep backend as:

```
artifacts/api-server
```

But ensure:

* It is NOT included in Vercel build
* It remains a separate deployable service (Railway/Render/Replit Deployments recommended)
* Remove any assumption that it runs in Vercel serverless environment unless explicitly converted

---

## 3. CREATE ROOT vercel.json (IMPORTANT)

Create a correct root-level `vercel.json` that ONLY deploys the frontend:

```json
{
  "version": 2,
  "projects": {
    "frontend": {
      "rootDirectory": "artifacts/investa-farm",
      "buildCommand": "pnpm run build",
      "outputDirectory": "dist"
    }
  }
}
```

If projects block is not supported, use this fallback format:

```json
{
  "version": 2,
  "builds": [
    {
      "src": "artifacts/investa-farm/package.json",
      "use": "@vercel/static-build",
      "config": {
        "distDir": "dist"
      }
    }
  ]
}
```
